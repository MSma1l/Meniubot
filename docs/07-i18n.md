# 07 — Multilingv (RO / RU)

Angajații văd totul în română sau rusă, după `User.language`. Panoul de administrare este
**doar în română**.

Nu există bibliotecă de i18n și nici fișiere de traduceri. Textele sunt dicționare Python și obiecte
JS, împrăștiate în **patru fișiere**. Asta e principala sursă de inconsistență din proiect.

## Unde trăiesc textele

### 1. `backend/bot.py` — mesajele botului

`TEXTS = {"ro": {...}, "ru": {...}}`, ~25 de chei. Acces prin `t(lang, key)`, cu fallback pe `ro`.

Acoperă: bun-venit, întrebarea despre nume, confirmarea înregistrării, reminderul, „mâncarea a
sosit", etichetele felurilor, și **ghidul complet** (cheia `guide`).

Excepții hardcodate, în afara dicționarului:
- mesajele de „bine ai revenit" din `start()` — `if lang == "ru": … else: …`
- `"⚠️ Serverul nu este disponibil momentan…"` — doar română
- `"Please /start first."` și `"Cancelled."` — doar engleză

### 2. `backend/app.py` — mesajele trimise de API

Cinci dicționare la nivel de modul:

| Dicționar | Trimis de |
|-----------|-----------|
| `FOOD_ARRIVED_TEXTS` | `POST /api/notify/food-arrived` |
| `SELECTION_CONFIRM_TEXTS` | `POST /api/selections` cu `source=webapp` |
| `SELECTION_NO_LUNCH_TEXTS` | idem, pentru `fara_pranz` |
| `ORDERING_CLOSED_TEXTS` | `POST /api/ordering/close` |
| `FEL_LABELS` | folosit în confirmarea selecției |

Plus etichete construite inline în `create_selection()`:
```python
f1_label = "Блюдо 1" if lang == "ru" else "Felul 1"
```

### 3. `backend/static/webapp/index.html` — Mini App

`const TEXTS = { ro: {...}, ru: {...} }`, ~28 de chei. Acces prin `tt(key)`, fallback pe `ro`.

Plus, în afara dicționarului:
- `formatDate()` — două tablouri de zile și două de luni
- salutul din `showWelcome()` — ternar `currentLang === 'ru' ? … : …`
- starea butonului la trimitere: `'⏳ Отправка...' : '⏳ Se trimite...'`
- alerta de eroare: `'Ошибка. Попробуйте снова.' : 'Eroare. Încercați din nou.'`

### 4. `backend/calculations.py` — raportul

`FEL_LABELS_RO` și `FEL_LABELS_RU`, folosite pentru cele două liste nominale.

Restul raportului (`TOTAL PORȚII`, `📊 DETALII COMENZI`, `MAXI (Felul 1 + Felul 2)`, `porții`)
este **hardcodat în română**. Doar antetul secțiunii ruse și numele meniurilor sunt traduse.

### 5. Baza de date — conținutul

Textele care se schimbă zilnic nu stau în cod:

| Tabel | Câmpuri RO | Câmpuri RU |
|-------|-----------|-----------|
| `menus` | `name`, `felul_1`, `felul_2`, `garnitura` | `name_ru`, `felul_1_ru`, `felul_2_ru`, `garnitura_ru` |
| `instructions` | `title`, `content` | `title_ru`, `content_ru` |

## Regula de fallback

**Pentru chei de cod:** lipsa unei chei în `ru` întoarce varianta `ro`. Nimic nu se rupe, dar
utilizatorul rus vede text românesc.

**Pentru conținut din bază:** câmpul `_ru` se folosește doar dacă limba e `ru` **și** câmpul e
nevid. Altfel se cade pe varianta română.

```python
mname = menu.name
if lang == "ru" and menu.name_ru:
    mname = menu.name_ru
```

Același tipar, în JS:
```js
function mName(m) { return (currentLang === 'ru' && m.name_ru) ? m.name_ru : m.name; }
```

Consecință: un meniu cu `name_ru` gol apare rusofonilor cu numele românesc. Nu e o eroare, e
comportamentul intenționat.

## Cele două ghiduri care pot diverge

Există **două** surse de „cum se comandă":

1. `TEXTS[lang]["guide"]` din `bot.py` — text static, trimis la `/guide` și după înregistrare.
   Conține ore hardcodate: „⏰ Program comenzi: 9:00 — 10:30".
2. Tabela `instructions` — pași cu imagini, editabili din panou, afișați la 📖 în Mini App.

Nimic nu le sincronizează. Dacă administratorul schimbă fereastra de remindere din Dashboard,
textul din `bot.py` rămâne cu orele vechi.

## Cum adaugi sau schimbi un text

Folosește skill-ul `/meniubot-i18n` — parcurge toate cele patru fișiere și verifică simetria
`ro` ↔ `ru`.

Manual, lista de verificat:

- [ ] Textul e **și** în `ro`, **și** în `ru`?
- [ ] Dacă e mesaj de bot trimis din API → `app.py`, dicționarul potrivit
- [ ] Dacă e mesaj de bot trimis din procesul bot → `bot.py`, `TEXTS`
- [ ] Dacă apare în Mini App → `static/webapp/index.html`, `TEXTS`
- [ ] Dacă apare în raport → `calculations.py`
- [ ] Dacă apare în panoul de admin → hardcodat în TSX, doar română
- [ ] Ai atins vreo oră sau adresă care e duplicată în `TEXTS[lang]["guide"]`?

## Adăugarea unei a treia limbi

README-ul menționează „RO/RU/EN". În cod **engleza nu există** — botul oferă doar două butoane de
limbă. Ar fi nevoie de: o cheie `"en"` în cele patru dicționare, coloane `_en` în `menus` și
`instructions` (plus migrație în `migrate_db()`), un al treilea buton în `start()`, și extinderea
raportului cu o a treia listă nominală.
